Q1: Discuss the scope of var, let, and const

var : Variables declared with var can be reassigned within the same scope. This means you can change their values after the initial assignment. They are also hoisted to the top of their scope, which can sometimes lead to unexpected behavior.

let : Variables declared with let can also be reassigned within the same scope. they are block-scoped, meaning they are limited to the block in which they are defined.

const :Variables declared with const cannot be reassigned after their initial assignment. They are also block-scoped. it's important to note that while the variable itself cannot be reassigned, the value it holds can still be mutable if it's an object or an array.

Q2: Tell us the use cases of null and undefined

null : null is used when the value does not exist, when you do not want to reject a default value.

undefined : undefined value does not exist, it Used when a variable is not set to a value or a function does not return a value.

Q3: What do you mean by REST API?

REST API : Representational State Transfer Application Programming Interface, is a set of guidelines and principles for building and interacting with web services using the HTTP protocol. It is a popular architectural style for designing networked applications and is widely used for creating APIs that allow different software systems to communicate with each other over the internet.
